AI Demo Workflow

This is an interactive workflow demonstration. The goal is to visually communicate the proposed website workflow between Smith Brothers and Delphic Digital. All of the content in this site is example only.

Proposed Workflow

Roles
UX Designer
Designer
Front-End Developer
Back-End Developer

Step 1
UX Designer works in tandem with Designer.

Wireframe Creation

Create annotated wireframes. Wireframes will also determine primary, secondary, and tertiary module development. They will attempt to be as detailed as possible to define module re-use, nesting, movement, and variations.

Style Tile Creation

Create style tiles.

Step 2
Front-End Developer works in tandem with the Designer.

Module & Style Guide Development

Develop base site styles (typography, lists, tables, etc.). Develop primary site modules (these are defined in the annotated wireframe). Secondary and tertiary modules will continue to be developed in the longer front-end development sprint. Code will be heavily commented and classes will be added to modules to account for component variations.

Module design

Design primary modules. Continue with secondary and tertiary modules in the longer design sprint.

Step 3
Front-end Developer works in tandem with the Designer and UX Designer.

HTML Prototype Development

Develop basic site pages for demonstration using the layouts defined in the annotated wireframes and the developed primary modules.

HTML Prototype Design

Design basic site pages for demonstration. Designer works heavily with the front-end developer and UX designer to collaborate and modify the HTML prototypes to fit the overall design better.

Step 4
Front-end Developer works in tandem with the Back-end Developer.

HTML Template Handoff

Front-end developer hands over HTML templates using Git.

Sitecore Integration

Back-end developer begins componentization based on annotated wireframes, module guide, and HTML prototypes.

Step 5
It takes a village. Client revisions. Restart flow.
